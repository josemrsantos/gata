# Implementation Plan: Text Output Bundle

**Branch**: `004-text-output-bundle` | **Date**: 2026-05-10 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/004-text-output-bundle/spec.md`

## Summary

After every pipeline run, produce a bundle folder containing two conversation logs (Agent 0
and B/C), an in-language HTML explanation, an English deep-dive HTML, and a plain-text prompt
card. Conversation logs come from a refactored `DualPersonaLoop` that captures all turns as
structured data. HTML generation is handled by a new `agent_explainer` dual-LLM agent
(Claude writer + Gemini editor) reusing the existing `<verdict>` tag protocol. The bundle
folder is written by a new `bundle_writer` module invoked by the pipeline after image
generation, with graceful degradation on partial failures.

## Technical Context

**Language/Version**: Python 3.12 (runtime), targeting >=3.10 per pyproject.toml
**Primary Dependencies**: anthropic, google-genai, python-dotenv, pyyaml, pytest, ruff
**Storage**: Local filesystem — bundle folder sibling to output PNG
**Testing**: pytest with mocked Anthropic and Gemini SDK calls
**Target Platform**: Linux local (same as existing pipeline)
**Project Type**: CLI pipeline
**Performance Goals**: Bundle write must complete within 5 seconds of pipeline failure (SC-005)
**Constraints**: No new API keys or external services — reuse existing clients
**Scale/Scope**: One bundle per pipeline run; single-user local operation
