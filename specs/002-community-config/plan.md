# Implementation Plan: Community Configuration

**Branch**: `002-stage-2-community-config` | **Date**: 2026-05-03 | **Spec**: specs/002-community-config/spec.md

## Summary

Replace hardcoded `TOPIC` and `BRIEF` constants in `pipeline.py` with a `communities.yaml` config file. Add
community-mode CLI (`--community <name>` or random selection) and manual-mode CLI (`--topic`, `--audience`,
`--language`, `--tone`). Migrate all bare `print()` calls in agent and pipeline code to Python's `logging` module
(Constitution Principle 13). Output images are saved to `output/{community-name}/{topic}.png` with both path segments
sanitized consistently.

## Key Design Decisions

### 1. Config loader as a separate module (`agents/config_loader.py`)

Keeps `pipeline.py` focused on orchestration. The loader is independently testable without invoking any agent or API.
Exposes two public functions: `load_communities(path: str) -> list[Community]` and
`sanitize_path_segment(text: str) -> str`.

### 2. `Community` dataclass in `agents/types.py`

Consistent with existing `StrategyBrief`, `CartoonConcept`, `Critique` pattern. Fields mirror the YAML schema exactly:
`name`, `target_audience`, `output_language`, `tone`, `topics`. `to_brief()` convenience method returns a
`StrategyBrief` — no translation code scattered across the pipeline.

### 3. Sanitization applied to both path segments

`sanitize_path_segment(text)` applies: `text.lower()`, spaces → underscores, strip non-`[a-z0-9_]`, truncate to 50
chars. Called for both `{community-name}` and `{topic}`. Single function, tested once, reused twice.

### 4. `agent_d.generate()` accepts `output_path: str` parameter

Removes the module-level `OUTPUT_PATH` constant. `pipeline.py` computes the full path from sanitized segments and
passes it in. Existing tests that patched `OUTPUT_PATH` are updated to pass the path directly — simpler and more
explicit.

### 5. Logging setup in `pipeline.py` only

`logging.basicConfig(level=logging.INFO, format="%(asctime)s [%(name)s] %(levelname)s %(message)s")` called once in
`main()`. Each module gets `logger = logging.getLogger(__name__)`. No logging configuration in agent modules — they
only obtain a logger, never configure one.

### 6. `argparse` for CLI

Stdlib, no new non-test dependency beyond `pyyaml`. Two mutually exclusive argument groups:
- Community group: `--community` (optional)
- Manual group: `--topic`, `--audience`, `--language`, `--tone` (all required together)

If neither group is provided → random community mode. If both groups are present → argparse conflict error before any
other code runs.

### 7. YAML library: `pyyaml`

De facto standard. Simple flat schema — no need for schema validation libraries (Pydantic, cerberus). Validation is
hand-written in `config_loader.py` and is fully testable. Added to `pyproject.toml` `dependencies`.
